# Restaurant Review Compass - 餐饮评论分类系统

## 项目简介

Restaurant Review Compass 是一个全面的中文餐饮评论文本分类系统，集成了多种机器学习和深度学习方法，用于将用户评价自动分类到多个预定义的情感极性（如正面、中性、负面）或业务属性（如菜品口味、服务态度、环境卫生）中。该项目展示了从传统机器学习到预训练模型再到大语言模型的完整技术演进路线。

## 分类类别

用于判定评论的紧急程度与业务影响。

## 技术方案

### 1. 数据处理 

- 数据集：训练集、验证集、测试集
- EDA分析：标签分布、文本长度统计等
- 预处理：停用词过滤、文本清洗

### 2. 随机森林

- 特征提取：TF-IDF向量化
- 模型：RandomForestClassifier
- 评估指标：Accuracy、Precision、Recall、F1-score

### 3. FastText

- 支持词级别和字符级别训练
- 提供默认参数和自动超参数调优两种模式
- 轻量级文本表示学习方法

### 4. BERT

- 模型：bert-base-chinese
- 特点：基于Transformer的预训练模型
- 训练：AdamW优化器 + 交叉熵损失
- 评估：在验证集上保存最佳模型

### 5. 大语言模型 

- 模型：qwen-plus
- 方法：零样本分类（Zero-shot Classification）
- API调用：OpenAI兼容接口

### 6. 模型压缩 

- **量化（Quantization）**：减少模型参数精度
- **知识蒸馏（Distillation）**：BERT → BiLSTM
- **剪枝（Pruning）**：移除不重要的神经元

## 评估指标

- **准确率 (Accuracy)**：分类正确的样本比例
- **精确率 (Precision)**：预测为正类的样本中真正为正类的比例
- **召回率 (Recall)**：实际为正类的样本中被正确预测的比例
- **F1分数**：精确率和召回率的调和平均

## 注意事项

1. 首次运行前请确保配置正确的API密钥
2. BERT模型训练需要GPU支持
3. 数据集路径需根据实际情况配置
4. 严禁惯性思维：不要默认 1=正，0=负。在本体系中，0专指负面，1 包含正面及未提及。
5. 未提及即1：对于文本中未出现的维度，senti_label 对应位必须填 1，切勿留空或填0。
6. 多标签顺序：aspect_label_id 中的ID顺序应与 senti_label 的维度顺序无关，但建议按ID升序排列以便核对。
7. 风险底线：只要出现“吃出异物、拉肚子、后厨脏乱”等描述，无论语气是否平和，必须标记为 3 (高危)。
8. 格式规范：输出结果需严格符合JSON格式，字段名不可更改。
